# L01: From Prompt Engineering to Context Engineering

## Overview

This lesson introduces the paradigm shift from prompt engineering -- writing better individual prompts -- to context engineering -- systematically managing everything the model sees. You will learn why context is a system, not just a string, and how this mindset change leads to dramatically better AI application outcomes.

## Topics Covered

- The limitations of prompt engineering as a standalone discipline
- Context engineering: managing the full input a model receives
- Components of context: system instructions, conversation history, retrieved data, tool outputs
- Context as a system: designing the interplay between context components
- The context window as a scarce resource requiring intentional allocation
- Practical examples of prompt engineering failures solved by context engineering
- Mental models for thinking about context design in production applications

## Key Takeaways

- Prompt engineering optimizes a single turn; context engineering optimizes the entire system
- The model's output quality is bounded by the quality and relevance of its full context
- Context is not just what you write -- it includes retrieved documents, tool outputs, and conversation history
- Treating context as a system with interacting components leads to more reliable AI applications
- The shift to context engineering reflects the maturation of AI application development as a discipline
