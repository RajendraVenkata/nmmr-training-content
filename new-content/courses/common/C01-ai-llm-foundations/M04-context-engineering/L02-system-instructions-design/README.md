# L02: System Instructions Design

## Overview

This lesson teaches the art and science of crafting effective system instructions. You will learn to define clear roles, specify constraints, enforce output formats, and structure system prompts that consistently guide model behavior across diverse user inputs.

## Topics Covered

- The role of system instructions in shaping model behavior
- Role definition: persona, expertise, and behavioral boundaries
- Constraint specification: what the model should and should not do
- Output formatting: enforcing structure, tone, length, and style
- Few-shot examples within system instructions
- Ordering and structuring system prompt sections for maximum impact
- Testing and iterating on system instructions
- Common anti-patterns and how to avoid them

## Key Takeaways

- Well-designed system instructions dramatically reduce output variability across user inputs
- Role definition should include not just who the model is, but explicit behavioral boundaries
- Constraints are more effective when stated positively ("always do X") rather than only negatively ("don't do Y")
- Output format specifications should include concrete examples of the expected structure
- System instructions should be tested against adversarial and edge-case inputs, not just happy-path scenarios
