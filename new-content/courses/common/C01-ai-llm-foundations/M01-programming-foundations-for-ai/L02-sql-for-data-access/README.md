# L02: SQL for Data Access

## Overview

This lesson builds advanced SQL skills required for AI data pipelines, including window functions, common table expressions, complex joins, and querying vector stores. These capabilities are essential for preparing training data, managing embeddings, and building retrieval systems.

## Topics Covered

- Window functions for ranking, running totals, and partitioned aggregations
- Common Table Expressions (CTEs) for readable, modular queries
- Complex joins across multiple tables and self-joins
- Querying vector stores for similarity search
- SQL patterns for AI data pipeline management
- Performance considerations for large-scale AI datasets

## Lab: Writing SQL Queries for AI Data Pipelines

Write a series of progressively complex SQL queries that simulate real AI data pipeline tasks, including preparing training datasets with window functions, managing embedding metadata with CTEs, and performing similarity searches against a vector store.

### Lab Objectives

- Write window functions to rank and partition AI training data
- Use CTEs to build modular, multi-step data transformation queries
- Construct complex joins to combine data from multiple sources for model input
- Query a vector store using SQL-based similarity search patterns
- Optimize query performance for large embedding tables

## Key Takeaways

- Window functions enable powerful analytical queries without collapsing result sets
- CTEs make complex AI data queries readable, testable, and maintainable
- Vector store querying via SQL is a foundational skill for RAG implementations
- Understanding query performance is critical when working with large embedding datasets
