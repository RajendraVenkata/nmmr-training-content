# LlamaIndex Workflows - Event-Driven Agents

## Overview

Learn LlamaIndex Workflows, an event-driven, asynchronous-first system where "steps" broadcast and receive "events" to handle complex, non-linear agent logic including failure recovery.

## Topics Covered

- Steps and events architecture
- Asynchronous-first design patterns
- Event broadcasting and receiving
- Retry events on failure for self-healing agents
- Non-linear workflow patterns

## Lab: Creating an Event-Driven Agent

Build an event-driven agent using LlamaIndex Workflows with asynchronous steps, event broadcasting, and failure recovery.

### Lab Objectives

- Build an agent using LlamaIndex Workflows step/event model
- Implement asynchronous event broadcasting between steps
- Add retry events for failure recovery
- Test non-linear execution flows

## Key Takeaways

- Event-driven architecture is ideal for complex, non-linear agent workflows
- Asynchronous-first design enables efficient handling of I/O-bound operations
- Retry events provide self-healing capabilities when individual steps fail
