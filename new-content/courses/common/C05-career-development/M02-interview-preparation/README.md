# M02: Interview Preparation

## Overview

This module prepares you for the full spectrum of AI job interviews, from technical screen to system design round. You will review core technical foundations, sharpen your data interpretation skills, learn a proven storytelling framework for presenting your work, and practice trade-off analysis for system design questions.

## Prerequisites

- Completion of M01 (Portfolio Building) or an existing portfolio of AI projects
- Working knowledge of Python, SQL, and basic statistics
- Familiarity with at least one AI framework or LLM API

## Lessons

| Lesson | Name | Has Lab? | Key Topics |
|--------|------|----------|------------|
| L01 | Technical Foundations Review | No | Python mastery (pandas, numpy, polars), SQL proficiency (window functions, CTEs), statistics (hypothesis testing, confidence intervals, bias vs variance), modern AI stack baseline |
| L02 | EDA & Data Interpretation Skills | No | Business framing, data integrity checks, leakage and bias detection, embedding visualization (UMAP, PCA), interpreting clusters |
| L03 | The Storytelling Framework | No | Problem-Data-Methods-Failures-Wins-Impact structure, explaining technical decisions to engineers and business stakeholders |
| L04 | Trade-off Analysis & System Design | No | Latency vs cost vs quality trade-offs, system design interview patterns for AI, architecture decision justification, mock interview practice |

## Key Tools

- Python (pandas, numpy, polars) for technical review
- SQL (window functions, CTEs, complex joins)
- Statistics fundamentals (hypothesis testing, confidence intervals)
- UMAP, PCA for embedding visualization
- Whiteboarding tools for system design practice

## Learning Outcomes

- Demonstrate fluency in Python data libraries, advanced SQL, and statistics during technical screens
- Perform exploratory data analysis with a business-first framing and identify data integrity issues
- Present technical projects using the Problem-Data-Methods-Failures-Wins-Impact storytelling structure
- Analyze and articulate latency, cost, and quality trade-offs in AI system design interviews
- Adapt your communication style to both engineering and business audiences
- Practice mock interview scenarios covering coding, data interpretation, and system design rounds
