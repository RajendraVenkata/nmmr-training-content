# L03: Open-Sourcing & Documentation

## Overview

This lesson covers how to make your AI projects visible and accessible to recruiters, hiring managers, and the broader developer community. You will learn the art of writing clear READMEs, documenting architecture decisions, and structuring open-source repositories so that your skills are tangible and discoverable.

## Topics Covered

- Making projects public on GitHub with proper licensing
- Writing clear, structured READMEs that communicate value quickly
- Documenting architecture decisions with Architecture Decision Records (ADRs)
- Creating setup and contribution guides for open-source projects
- Using diagrams, screenshots, and demos to showcase project functionality
- Making skills tangible and visible to recruiters who skim repositories
- GitHub profile optimization and pinned repository strategy

## Key Takeaways

- A well-documented public repository is worth more than a private project with no visibility
- READMEs should answer "what does this do, why does it matter, and how do I run it" within the first scroll
- Architecture Decision Records make your technical judgment visible and demonstrate senior-level thinking
- Screenshots, diagrams, and demo GIFs dramatically increase the chance a recruiter or engineer will engage with your project
- Your GitHub profile is a living portfolio -- pinned repos, contribution history, and documentation quality all send signals to employers
