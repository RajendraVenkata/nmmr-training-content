# M01: Portfolio Building

## Overview

This module teaches you how to build an AI portfolio that stands out to hiring managers and recruiters. You will learn why production-grade projects matter more than tutorial demos, which five project types carry the most weight, and how to document and open-source your work for maximum visibility.

## Prerequisites

- At least one completed AI project (from any prior course or personal work)
- A GitHub account with basic proficiency in repositories and commits
- Familiarity with writing Markdown files

## Lessons

| Lesson | Name | Has Lab? | Key Topics |
|--------|------|----------|------------|
| L01 | From Demos to Production-Grade Projects | No | Why hiring managers skip chatbot tutorials, system-level thinking, demonstrating production reliability and measurable outcomes |
| L02 | The Five Portfolio Projects That Matter | No | Production-grade RAG system, Local AI Assistant (SLM), Monitoring & Observability Layer, Specialized Fine-Tuning Project, Real-Time Multimodal Application |
| L03 | Open-Sourcing & Documentation | No | Making projects public on GitHub, writing clear READMEs, documenting architecture decisions, making skills visible to recruiters |

## Key Tools

- GitHub for hosting and showcasing projects
- README templates and badges
- Architecture diagramming tools (Mermaid, Excalidraw)
- Markdown for documentation

## Learning Outcomes

- Distinguish between demo-quality and production-grade AI projects from a hiring manager's perspective
- Identify the five portfolio project types that demonstrate the strongest signal to employers
- Restructure existing projects to emphasize system-level thinking, reliability, and measurable business outcomes
- Write clear, structured READMEs that communicate project value in under 60 seconds
- Document architecture decisions and trade-offs to make technical judgment visible to reviewers
