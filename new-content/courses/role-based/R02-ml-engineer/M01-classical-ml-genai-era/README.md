# M01: Classical ML in the GenAI Era

## Overview

Revisit classical machine learning fundamentals through the lens of modern AI workflows. This module covers feature engineering with leakage prevention, reproducible training pipelines with experiment tracking, and model serving patterns for both batch and real-time inference.

## Prerequisites

- Strong foundation in supervised and unsupervised machine learning
- Proficiency in Python, scikit-learn, and pandas
- Familiarity with basic model evaluation metrics

## Lessons

| Lesson | Name | Has Lab? | Key Topics |
|--------|------|----------|------------|
| L01 | Feature Engineering & Leakage Prevention | No | Feature engineering best practices, data leakage identification, feature importance, temporal leakage |
| L02 | Training Pipelines & Experiment Tracking | Yes | Reproducible pipelines, W&B integration, model versioning, hyperparameter management |
| L03 | Model Serving - Batch vs Real-Time | No | Batch processing, real-time inference, latency requirements, serving architecture patterns |

## Key Tools

- scikit-learn, XGBoost, LightGBM
- Weights & Biases (W&B)
- MLflow
- pandas, NumPy
- Docker for serving

## Learning Outcomes

- Apply feature engineering best practices while identifying and preventing data leakage
- Build reproducible training pipelines with full experiment tracking and model versioning
- Choose the appropriate model serving pattern (batch vs. real-time) based on latency and throughput requirements
