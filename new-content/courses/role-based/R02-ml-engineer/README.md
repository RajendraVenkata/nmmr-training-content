# R02: ML Engineer Specialization

## Overview

The classical ML role is evolving to integrate generative AI into traditional ML workflows. This course equips ML engineers with skills to bridge classical machine learning and LLM-powered systems, covering hybrid ML-LLM architectures, data-centric iteration, and modern experiment tracking.

## Prerequisites

- Beginner course track (B01-B10) completed
- Intermediate course track (I01-I03) completed
- Strong foundation in classical machine learning (supervised/unsupervised learning, model evaluation)
- Proficiency in Python, scikit-learn, and pandas

## Target Role

ML Engineers design and maintain machine learning pipelines that increasingly incorporate LLMs as components. They bridge the gap between classical ML (feature engineering, model training, serving) and generative AI (LLMs as feature generators, weak labelers, and judges). This role requires strong fundamentals in data science, experiment tracking, and production ML systems.

## Modules

| Module | Name | Lessons | Key Topics |
|--------|------|---------|------------|
| M01 | Classical ML in the GenAI Era | 3 | Feature engineering, training pipelines, experiment tracking, model serving |
| M02 | Hybrid ML-LLM Systems | 3 | LLMs as feature generators, weak labelers, and judges for ML outputs |
| M03 | Data-Centric Iteration | 2 | Labeling strategies, active learning, data quality, pipeline management |

## Key Tools & Technologies

- scikit-learn, XGBoost, LightGBM
- Weights & Biases (W&B) for experiment tracking
- LLM APIs (OpenAI, Anthropic) for hybrid workflows
- Label Studio or Prodigy for annotation
- Apache Airflow / Prefect for pipeline orchestration
- MLflow for model versioning

## Learning Outcomes

- Apply feature engineering best practices while preventing data leakage in ML pipelines
- Set up reproducible training pipelines with experiment tracking and model versioning
- Design hybrid systems that use LLMs as feature generators, weak labelers, and quality judges
- Implement active learning loops for efficient data labeling and model improvement
- Build and manage data quality pipelines with drift detection and quality gates
