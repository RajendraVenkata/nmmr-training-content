# M03: Data-Centric Iteration

## Overview

Shift focus from model-centric to data-centric approaches for improving ML systems. This module covers building effective labeling strategies with active learning, managing data quality pipelines, and implementing data drift detection for production systems.

## Prerequisites

- R02-M01: Classical ML in the GenAI Era
- R02-M02: Hybrid ML-LLM Systems
- Understanding of data pipeline concepts

## Lessons

| Lesson | Name | Has Lab? | Key Topics |
|--------|------|----------|------------|
| L01 | Labeling Strategies & Active Learning | Yes | Labeling strategies, active learning loops, sample selection, annotation guidelines |
| L02 | Data Quality & Pipeline Management | No | Data quality constraints, pipeline orchestration, data lineage, drift detection |

## Key Tools

- Label Studio / Prodigy for annotation
- Apache Airflow / Prefect for orchestration
- Great Expectations for data quality
- scikit-learn for active learning
- pandas for data analysis

## Learning Outcomes

- Design and implement active learning loops that maximize labeling efficiency
- Build data quality pipelines with automated validation and drift detection
- Manage data lineage and provenance across complex ML pipelines
- Apply data-centric iteration principles to systematically improve model performance
