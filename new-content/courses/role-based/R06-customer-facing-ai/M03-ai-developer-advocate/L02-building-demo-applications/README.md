# L02: Building Demo Applications (Gradio/Streamlit)

## Overview

Learn how to create compelling, interactive demo applications that make complex AI capabilities accessible and understandable. This lesson covers Gradio and Streamlit frameworks, design principles for effective demos, and techniques for turning technical concepts into experiences that resonate with developers and stakeholders.

## Topics Covered

- Creating compelling and polished demo applications
- Gradio for ML demos: components, layouts, and interactivity
- Streamlit for data apps: widgets, state management, and deployment
- Making complex AI accessible through interactive interfaces
- Design principles for effective demos (simplicity, speed, wow factor)
- Deploying demos for public access (Hugging Face Spaces, Streamlit Cloud)

## Lab: Building a Polished Demo Application

Build a polished, interactive demo application using Gradio or Streamlit that showcases an AI capability in a way that is immediately understandable and engaging for a technical audience.

### Lab Objectives

- Design a demo application that clearly communicates an AI capability
- Implement the demo using Gradio or Streamlit with clean UI and responsive interactions
- Handle edge cases and errors gracefully in the demo interface
- Deploy the demo to a public hosting platform for sharing

## Key Takeaways

- The best demos are simple, fast, and immediately demonstrate value without requiring explanation
- Gradio excels at ML-specific interfaces (input/output pairs) while Streamlit is better for data exploration and multi-page apps
- Error handling in demos is critical; a crash during a presentation destroys credibility
- Demo design should prioritize the "aha moment" that makes viewers understand the technology's potential
