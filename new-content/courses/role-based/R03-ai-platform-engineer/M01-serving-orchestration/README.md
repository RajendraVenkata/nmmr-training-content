# M01: Serving & Orchestration

## Overview

Master the fundamentals of deploying and orchestrating AI workloads in production. This module covers container orchestration for AI-specific requirements, autoscaling and reliability patterns with SLO-driven design, and secure secret management for AI systems.

## Prerequisites

- Working knowledge of Docker and container concepts
- Basic Kubernetes understanding
- Familiarity with cloud infrastructure (AWS, Azure, or GCP)

## Lessons

| Lesson | Name | Has Lab? | Key Topics |
|--------|------|----------|------------|
| L01 | Container Orchestration for AI | Yes | Docker for AI, Kubernetes, GPU-aware scheduling, container optimization |
| L02 | Autoscaling & Reliability Patterns (SLOs) | No | SLO definition, autoscaling strategies, load testing, circuit breakers |
| L03 | Secret Management & Access Control | No | API key management, secret rotation, access control, PII handling |

## Key Tools

- Docker
- Kubernetes (with GPU plugins)
- Helm charts
- Prometheus / Grafana
- HashiCorp Vault
- Load testing tools (Locust, k6)

## Learning Outcomes

- Deploy AI models in containerized environments with GPU-aware scheduling
- Define SLOs for AI systems and implement autoscaling strategies that meet them
- Set up reliable AI serving infrastructure with circuit breakers and graceful degradation
- Implement secure secret management and access control for API keys and sensitive data
